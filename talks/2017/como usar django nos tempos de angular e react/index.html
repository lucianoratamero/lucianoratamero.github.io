três pontos de vista

  • ponto de vista da empresa

três tipos de abordagem

  • moderna: react/angular/vue/... + API (DRF)
  • tradicional: django + JS embutido
  • híbrida: django + JS desacoplado

características da abordagem moderna

  • produto com cara de app
  • rápida prototipagem de interface
  • baixo custo a curto prazo, com aumento linear
  • fácil gerência de equipes
  • difícil coordenação de tarefas
  • decisões prévias
  • alta aposta no produto

características da abordagem tradicional

  • produto com cara de web
  • rápida prototipagem em geral
  • baixo custo, com aumento exponencial
  • gerência complicada de equipes => feudos
  • coordenação complicada de tarefas => monolítico
  • baixa aposta no produto

características da abordagem híbrida

  • produto com a cara que quiser
  • velocidade dependente da equipe e do momento
  • custo maior a curto prazo, dependente da equipe a longo
  • fácil gerência de equipes
  • fácil gerência de tarefas
  • aposta flexível no produto

maiores dependências

moderna: gerência

tradicional: produto

híbrida: equipe

moderna vai bem com:

  • startups
  • equipes pequenas/remotas
  • proximidade de domínio entre devs e gerentes

tradicional vai bem com:

  • equipes/produtos estáticos
  • projetos pessoais/descartáveis/burocráticos
  • fábricas de software

híbrida vai bem com:

  • equipes multidisciplinares
  • projetos longos, de baixa iteração
  • projetos com grande backlog
  • equipes desacopladas do cliente

seja onde for

  • não use o framework "porque sim"
  • deixe as decisões importantes pro futuro
  • encontre seu fluxo ideal (se puder)
  • experimente
